처방 요청 전체 흐름 (v5)
1. 사용자가 /mylab에서 처방 생성 요청
   Vue → POST /api/formula/generate
   (httpOnly Cookie JWT 자동 첨부)

2. VPS Hono API 수신
   → JWT 검증 (users.db 확인)
   → cf-tunnel → CF Workers 전달

3. CF Workers 캐시 확인
   ① KV 완전 일치 → 0.1초 반환
   ② KV 유사 처방 → 0.3초 반환
   ③ 캐시 미스 → PC 서버로 전달

4. PC 서버 처방 생산 (2~3초)
   Orchestrator → Agent 01~07
   → SQLite DB 조회 (외부 API 없음)
   → 처방서 + 전성분표 JSON 반환

5. VPS → CF KV 캐시 저장 (24h)
   → formula.db에 user_id와 함께 저장
   → Vue에 결과 반환

6. PC 오프라인 시 Fallback
   VPS → "처방 생성 대기중" 응답
   → n8n이 감지 → Hiworks 알림
   → PC 복구 후 자동 처리
VPS Nginx 설정 (coching.kr)
# /etc/nginx/sites-available/coching.kr
server {
  listen 80;
  server_name coching.kr www.coching.kr;
  # HTTPS 리다이렉트
  return 301 https://$host$request_uri;
}

server {
  listen 443 ssl;
  server_name coching.kr www.coching.kr;
  
  # SSL (Let's Encrypt 무료)
  ssl_certificate /etc/letsencrypt/live/coching.kr/fullchain.pem;
  ssl_certificate_key /etc/letsencrypt/live/coching.kr/privkey.pem;

  # Vue 앱 서빙 (기존 + 마이랩 통합 빌드)
  location / {
    root /var/www/coching/dist;
    try_files $uri $uri/ /index.html;
  }

  # VPS 인증 API
  location /api/auth/ {
    proxy_pass http://localhost:3001;
    proxy_set_header Authorization $http_authorization;
  }

  # 처방 API → CF Tunnel → PC 서버
  location /api/formula/ {
    proxy_pass http://localhost:3001;
    # CF Workers를 거쳐 PC로 전달
  }
}

Vue 로그인 플로우
// stores/auth.ts (Pinia)
export const useAuthStore = defineStore('auth', () => {
  const user = ref(null)

  const login = async (email, password) => {
    const res = await fetch('/api/auth/login', {
      method: 'POST',
      credentials: 'include',   // httpOnly Cookie 자동 처리
      body: JSON.stringify({ email, password })
    })
    if (res.ok) {
      await fetchMe()              // 사용자 정보 로드
      router.push('/mylab')
    }
  }

  const fetchMe = async () => {
    const res = await fetch('/api/auth/me', {
      credentials: 'include'     // Cookie 자동 첨부
    })
    user.value = await res.json()
  }

  return { user, login, fetchMe }
})
📓 마이랩 기능 설계 처방 업무일지 플랫폼
처방 전문가의 업무 흐름 전체를 지원하는 4가지 핵심 기능입니다. 모두 MVP에 포함됩니다.
핵심 기능 1 — 날짜별 처방 업무 일지
날짜별로 처방 작업을 기록합니다. 오늘 만든 처방, 어제 수정한 처방을 시간순으로 확인할 수 있습니다.
// formula.db 쿼리 (PC 서버)
SELECT * FROM formulas
WHERE user_id = ?
ORDER BY created_at DESC
-- 결과: 날짜별 그룹핑으로 표시
-- 2026.03.10: SPF50+ 선크림
-- 2026.03.09: 나이아신아마이드 세럼
핵심 기능 2 — 처방에 메모/노트 추가
각 처방에 자유 메모를 추가합니다. "고객 A 요청", "SPF 시험 의뢰 예정", "벤토나이트 함량 조정 필요" 등 업무 맥락을 기록합니다.
// formula.db 스키마
CREATE TABLE formulas (
  id           TEXT PRIMARY KEY,
  user_id      TEXT,
  title        TEXT,
  product_type TEXT,
  status       TEXT,     -- draft/review/done
  formula_data TEXT,     -- JSON (처방 전체)
  memo         TEXT,     -- ← 메모/노트
  tags         TEXT,     -- JSON ["선크림","A사"]
  project_name TEXT,
  created_at   TEXT,
  updated_at   TEXT
);
핵심 기능 3 — 처방 진행 상태 관리
처방의 진행 상태를 3단계로 관리합니다. 상태 변경 히스토리도 기록되어 언제 검토가 완료됐는지 추적 가능합니다.
📝 초안 (draft)
AI 처방 생성 완료. 아직 검토 전 상태.
🔍 검토중 (review)
성분 검토, 규제 확인, 수동 수정 중.
✅ 완료 (done)
최종 확정. 제조 의뢰 가능 상태.
핵심 기능 4 — 처방을 프로젝트 단위로 관리
관련 처방을 프로젝트로 묶어 관리합니다. "A사 썬케어 라인"에는 SPF30, SPF50, PA+++ 처방 3개가 묶여 있는 식입니다.
// 프로젝트 뷰 쿼리
SELECT
  project_name,
  COUNT(*) as formula_count,
  SUM(CASE WHEN status='done' THEN 1 ELSE 0 END) as done_count
FROM formulas
WHERE user_id = ?
GROUP BY project_name
ORDER BY MAX(updated_at) DESC

-- "A사 썬케어 라인": 3개 처방 (2개 완료)
-- "B사 기초 세트": 5개 처방 (전체 완료)
